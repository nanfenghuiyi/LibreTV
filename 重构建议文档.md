# LibreTV 重构建议文档

> 约束：部署方式不变，功能不变。本文档仅提供架构与代码层面的重构方向。

---

## 一、现状诊断

### 1.1 代码规模与分布

| 文件 | 行数 | 问题 |
|------|------|------|
| `js/app.js` | ~1988 | 混合搜索、渲染、历史记录、配置、DOM 操作 |
| `js/player.js` | ~1792 | 混合播放器控制、集数渲染、资源切换、HTML 模板生成 |
| `js/ui.js` | ~975 | 混合 Toast、Loading、面板、弹窗、设置逻辑 |
| `js/api.js` | ~636 | 混合请求、数据解析、特殊源适配 |
| `js/douban.js` | ~793 | 混合豆瓣 API、标签管理、渲染、存储 |
| `css/styles.css` | ~1029 | 所有全局样式混在一处 |

### 1.2 核心问题

1. **全局命名空间污染**：所有变量和函数暴露在 `window` 上，无模块边界，任何文件都可修改全局状态。
2. **职责高度耦合**：单文件承担数据请求、业务逻辑、DOM 渲染、事件绑定多层职责。
3. **代码重复**：密码弹窗在 `index.html` 和 `player.html` 中重复定义约 60 行；XSS 转义代码 (`replace(/</g, '&lt;')`) 散落在 10 余处。
4. **存储管理混乱**：`localStorage` key 硬编码分散在各文件（`passwordVerified`、`proxyAuthHash`、`yellowFilterEnabled`、`doubanEnabled` 等），无统一管理。
5. **事件处理不统一**：大量使用内联 `onclick="func()"`，与部分 `addEventListener` 混用，难以维护和测试。
6. **代理层逻辑重复**：Vercel (`api/proxy/[...path].mjs`)、Express (`server.mjs`)、Netlify (`functions/_middleware.js`) 三处代理存在重复鉴权和请求转发逻辑。
7. **配置与逻辑混杂**：`js/config.js` 中定义了配置常量，又包含 `extendAPISites()` 等运行时逻辑。
8. **CSS 缺乏组件化**：`styles.css` 超过 1000 行，无明确分层；Tailwind 工具类与自定义 CSS 混用，维护成本高。

---

## 二、重构原则

1. **零构建工具**：保持纯静态文件部署，不引入 Webpack/Vite/Rollup 等构建流程。
2. **原生 ES Modules**：使用 `<script type="module">` + `export`/`import` 替代全局脚本加载，彻底隔离命名空间。
3. **分层架构**：按"配置层 → 数据层 → 业务层 → UI 层"纵向拆分，禁止跨层直接调用。
4. **单一职责**：单文件不超过 300 行，单一函数不超过 50 行。
5. **防御式渲染**：所有动态 HTML 必须通过统一的 `escapeHtml()` 生成，禁止裸用 `innerHTML` 拼接用户数据。

### 3.2 分层调用规则

```
pages/          →  orchestrate  → 调用 services + ui/components
services/       →  compose      → 调用 api/ + core/storage
api/            →  fetch        → 调用 core/config（读取 endpoint）
ui/components/  →  render       → 调用 core/dom + core/utils
```

**禁止**：
- `api/` 直接操作 `localStorage`
- `ui/` 直接调用 `fetch`
- `services/` 直接生成 HTML 字符串（应返回数据，由 `ui/renderers/` 渲染）

## 四、具体重构方案

### 4.1 模块拆分（重点）

#### 4.1.1 `js/app.js` → 拆分为 8+ 个模块

| 原 app.js 中的内容 | 目标文件 | 职责 |
|-------------------|---------|------|
| 全局变量（`selectedAPIs`、`customAPIs` 等） | `services/settings-service.js` | 状态管理 |
| `initAPICheckboxes()`、`renderCustomAPIsList()` | `ui/renderers/api-checkboxes.js` | 渲染 |
| `search()`、`handleApiRequest()` 调用 | `services/search-service.js` | 业务编排 |
| `renderResults()`、`renderTypeFilters()` | `ui/renderers/search-results.js` | 结果渲染 |
| 排序逻辑（`handleSortChange`、`sortResults`） | `services/search-service.js` | 数据转换 |
| 观看历史（`addToHistory`、`renderHistory`） | `services/history-service.js` + `ui/renderers/history-list.js` | 分离读写与渲染 |
| 配置导入导出 | `services/settings-service.js` | 序列化/反序列化 |
| DOMContentLoaded 初始化 | `ui/pages/home.js` | 页面入口 |

#### 4.1.2 `js/player.js` → 拆分为 5+ 个模块

| 原内容 | 目标文件 |
|--------|---------|
| ArtPlayer 初始化与控制 | `ui/components/art-player.js` |
| 集数列表渲染与倒序 | `ui/renderers/episode-list.js` |
| 资源切换逻辑 | `services/source-service.js` |
| 视频详情获取 | `api/vod.js` |
| 播放历史自动记录 | `services/history-service.js` |

#### 4.1.3 `js/api.js` → 拆分为 4 个模块

| 原内容 | 目标文件 |
|--------|---------|
| `fetch()` 封装 + 代理鉴权拼接 | `api/client.js` |
| 搜索/详情请求 | `api/vod.js` |
| 特殊源详情处理（`handleSpecialSourceDetail`） | `api/parser.js` |
| 数据格式化（添加 `source_name` 等） | `api/parser.js` |

#### 4.1.4 `js/ui.js` → 拆分为组件库

| 原内容 | 目标文件 |
|--------|---------|
| `showToast()` / Toast 队列 | `ui/components/toast.js` |
| `showLoading()` / `hideLoading()` | `ui/components/loading.js` |
| `toggleSettings()`、`toggleHistory()` | `ui/pages/home.js`（事件绑定） |
| 设置面板内的表单逻辑 | `services/settings-service.js`（数据）+ `ui/renderers/settings-panel.js`（渲染） |

### 4.2 配置层重构

**当前问题**：`config.js` 同时包含常量、运行时函数和全局挂载。

**目标**：

```javascript
// js/core/config.js
export const CONFIG = Object.freeze({
  PROXY_URL: '/proxy/',
  SEARCH_HISTORY_KEY: 'videoSearchHistory',
  MAX_HISTORY_ITEMS: 5,
  API_TIMEOUT: 10000,
  // ... 其他常量
});

export const SITE_CONFIG = Object.freeze({
  name: 'LibreTV',
  version: '1.1.0',
  // ...
});

export const PLAYER_CONFIG = Object.freeze({
  autoplay: true,
  timeout: 15000,
  // ...
});
```

- 使用 `Object.freeze()` 防止运行时篡改
- 移除 `window.API_SITES = API_SITES` 等全局挂载
- 运行时扩展（`extendAPISites`）移至 `services/source-service.js`

### 4.3 存储层重构

**当前问题**：`localStorage` key 分散在各处，无命名空间，易冲突。

**目标**：统一封装。

```javascript
// js/core/storage.js
const PREFIX = 'ltv:';

export const StorageKeys = Object.freeze({
  SELECTED_APIS: 'selectedAPIs',
  CUSTOM_APIS: 'customAPIs',
  HISTORY: 'history',
  SETTINGS: 'settings',
  AUTH_HASH: 'authHash',
  // ...
});

export const storage = {
  get(key, defaultValue = null) {
    try {
      const raw = localStorage.getItem(PREFIX + key);
      return raw ? JSON.parse(raw) : defaultValue;
    } catch {
      return defaultValue;
    }
  },
  set(key, value) {
    localStorage.setItem(PREFIX + key, JSON.stringify(value));
  },
  remove(key) {
    localStorage.removeItem(PREFIX + key);
  },
  clear() {
    Object.keys(localStorage)
      .filter(k => k.startsWith(PREFIX))
      .forEach(k => localStorage.removeItem(k));
  }
};
```

所有业务代码通过 `storage.get(StorageKeys.SELECTED_APIS)` 读写，禁止直接调用 `localStorage`。

### 4.4 事件处理重构

**当前问题**：大量内联 `onclick="func()"`。

**目标**：统一使用事件委托。

```javascript
// ui/pages/home.js
import { searchService } from '../../services/search-service.js';

document.addEventListener('DOMContentLoaded', () => {
  // 搜索按钮
  document.getElementById('searchBtn').addEventListener('click', () => {
    searchService.execute(getSearchQuery());
  });

  // 回车搜索
  document.getElementById('searchInput').addEventListener('keydown', (e) => {
    if (e.key === 'Enter') searchService.execute(getSearchQuery());
  });

  // 事件委托：结果卡片点击
  document.getElementById('results').addEventListener('click', (e) => {
    const card = e.target.closest('[data-vod-id]');
    if (card) {
      openDetailModal(card.dataset.vodId, card.dataset.source);
    }
  });
});
```

### 4.5 HTML 模板重构

**当前问题**：密码弹窗在 `index.html` 和 `player.html` 中重复。

**目标**：提取为独立 HTML 片段，由 JS 动态加载；或统一用 JS 组件生成。

```javascript
// ui/components/password-modal.js
export function createPasswordModal() {
  const div = document.createElement('div');
  div.id = 'passwordModal';
  div.className = 'fixed inset-0 bg-black/95 hidden items-center justify-center z-[65]';
  div.innerHTML = `
    <div class="bg-[#111] p-8 rounded-lg w-11/12 max-w-md border border-[#333]">
      <h2 class="text-2xl font-bold gradient-text">访问验证</h2>
      <form id="passwordForm">
        <input type="password" id="passwordInput" class="..." placeholder="密码...">
        <button type="submit" class="...">提交</button>
      </form>
    </div>
  `;
  document.body.appendChild(div);
  return div;
}
```

### 4.6 XSS 防护统一

**当前问题**：手动转义散落在各处。

**目标**：统一工具函数，渲染器强制使用。

```javascript
// js/core/utils.js
export function escapeHtml(str) {
  if (str == null) return '';
  return String(str)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

// ui/renderers/search-results.js
function renderCard(item) {
  return `
    <div class="card">
      <h3>${escapeHtml(item.vod_name)}</h3>
      <p>${escapeHtml(item.vod_remarks)}</p>
    </div>
  `;
}
```

### 4.7 CSS 重构

**当前问题**：`styles.css` 1029 行，无分层。

**目标**：

1. **变量化**：提取颜色和尺寸变量到 `css/base.css`。
   ```css
   :root {
     --bg-primary: #111;
     --bg-secondary: #151515;
     --border-color: #333;
     --accent-gradient: linear-gradient(to right, #6366f1, #a855f7, #ec4899);
   }
   ```

2. **组件化**：按组件拆分文件。
   - `css/components/modal.css`：所有弹窗基础样式
   - `css/components/cards.css`：视频卡片样式
   - `css/components/toast.css`：Toast 样式

3. **页面隔离**：页面特有样式只加载在对应页面。
   - `index.html` 加载 `css/pages/home.css`
   - `player.html` 加载 `css/pages/player.css`

4. **Tailwind 补充**：`utilities.css` 只放 Tailwind 未覆盖的自定义工具类，而非全部样式。

### 4.8 服务端重构

**当前问题**：
- `server.mjs` 中 `sha256Hash()` 用 `Promise` 包装同步操作，无意义。
- 三处代理（Express、Vercel、Netlify）存在重复鉴权逻辑。

**目标**：

1. **Express 精简**：移除无意义 Promise 包装。
   ```javascript
   // 之前
   function sha256Hash(input) {
     return new Promise((resolve) => {
       const hash = crypto.createHash('sha256');
       hash.update(input);
       resolve(hash.digest('hex'));
     });
   }

   // 之后
   function sha256Hash(input) {
     return crypto.createHash('sha256').update(input).digest('hex');
   }
   ```

2. **提取共享代理逻辑**：新建 `shared/proxy-core.mjs`，包含 `isValidUrl()`、`validateAuth()`、`fetchWithRetry()` 等通用函数。
   - `server.mjs` 和 `api/proxy/[...path].mjs` 均导入使用。
   - 注意：Netlify `functions/_middleware.js` 为 Edge 运行时，与 Node API 有差异，可保持独立或做适配层。

3. **配置收敛**：环境变量读取统一在 `shared/config.mjs`，避免各文件重复 `process.env.XXX`。

## 五、实施路线图

### 阶段一：基础设施（低风险，优先）

1. 新建 `js/core/` 目录，编写 `config.js`、`constants.js`、`storage.js`、`utils.js`、`dom.js`。
2. 新建 `css/base.css`，提取 CSS 变量；拆分 `css/components/` 和 `css/pages/`。
3. 验证 ES Module 在各部署目标（Express、Vercel、Netlify）上的兼容性。

### 阶段二：数据层与业务层（中风险）

1. 新建 `js/api/client.js`，将 `fetch` + 代理鉴权逻辑从 `api.js` 和 `proxy-auth.js` 迁移至此。
2. 新建 `js/services/settings-service.js`，将所有 `localStorage` 读写迁移至此。
3. 新建 `js/services/search-service.js`，将搜索聚合、排序、过滤逻辑从 `app.js` 迁移至此。
4. 确保迁移后的模块通过 `export`/`import` 引用，不再依赖全局变量。

### 阶段三：UI 组件化（中风险）

1. 提取 `ui/components/toast.js`、`loading.js`、`modal.js`、`password-modal.js`。
2. 提取 `ui/renderers/search-results.js`、`episode-list.js`、`history-list.js`。
3. 将 `index.html` 和 `player.html` 中的内联 `onclick` 替换为事件委托绑定。
4. 删除重复的密码弹窗 HTML，改为由 JS 组件动态创建。

### 阶段四：页面入口重构（中高风险）

1. 重写 `js/app.js` 为 `js/ui/pages/home.js`，仅负责页面初始化和事件绑定。
2. 重写 `js/player.js` 为 `js/ui/pages/player.js` + `js/ui/components/art-player.js`。
3. 更新 `index.html`、`player.html`、`watch.html` 的 `<script>` 标签为 `type="module"` 入口。

### 阶段五：服务端统一（低风险）

1. 新建 `shared/proxy-core.mjs` 和 `shared/config.mjs`。
2. 重构 `server.mjs` 和 `api/proxy/[...path].mjs`，导入共享逻辑。
3. 清理冗余代码和无意义 Promise 包装。

### 阶段六：清理与验证（低风险）

1. 删除旧的 `js/app.js`、`js/player.js`、`js/ui.js`、`js/api.js`、`js/douban.js`（确认功能完全迁移后）。
2. 全量功能回归测试（搜索、播放、历史、设置、豆瓣、自定义 API、密码验证、代理）。
3. 在各目标平台（本地 Express、Vercel、Netlify）部署验证。

## 六、兼容性保障

1. **浏览器兼容性**：原生 ES Modules 被所有现代浏览器支持（Chrome 61+、Firefox 60+、Safari 10.1+、Edge 16+）。项目目标用户群体满足此条件。
2. **服务端兼容性**：`server.mjs` 已使用 ES Module，无需改动。Vercel 和 Netlify Functions 已使用 `import`/`export`，兼容。
3. **回滚策略**：每个阶段独立提交 Git，确保单阶段出问题可快速回退。
4. **数据兼容性**：`storage.js` 封装层需兼容旧版 `localStorage` key（无 `ltv:` 前缀的遗留数据），首次读取时做迁移。
